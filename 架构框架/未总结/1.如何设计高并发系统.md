**高并发的哲学就是：找出单点、进行拆分**。将一个 “大单点” 拆成 “一个小单点+多个资源并行” 的形式。

>单点：一个 Web 系统中，几乎所有的流量都必须经过的特定部分，如果他出现瓶颈或故障，将导致整个系统的性能下降或不可用。单点是一个逻辑概念，在系统架构层面他可能指代各种不同层面的软件、硬件实体。

高并发又可以分为：计算高并发、网络高并发、存储高并发。

### 一、高并发架构的设计

**第一步：动静分离**

举一个例子：cpu：i5双核、内存：4g。使用一张静态图片测试 Apache、Nginx 的性能差距：两万对八万。

动态请求可以继续使用 http 协议发送给 apache；然后使用 nginx 承载静态资源的分发，静态资源也可以交给云服务商的 CDN 来承载。
